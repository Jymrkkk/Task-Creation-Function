Manual Test: Complete Create Flow

PENDING

Task: 15.1 Test complete create flow

Purpose: Verify that the complete task creation flow works end-to-end, including form submission, success modal, form persistence, history display, and "Create Another Task" functionality.

Requirements Validated: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 2.4, 3.1, 3.3, 3.4

Test Section 1: Submit New Task

Step 1.1: Open the Application

Open index.html in your browser.

Expected: The task creation form should be visible with all input fields empty (or default values).

Step 1.2: Fill in Task Details

Enter the following test data:

  • Task Name: "Test Task - Complete Create Flow"
  • Description: "This is a test task to verify the complete create flow functionality."
  • Priority: High
  • Assigned To: Development Team
  • Created By: "Test User"
Expected: All fields should accept input and display the entered values.

Step 1.3: Submit the Task

Click the "Submit Task" button.

Expected:
  • Button should show "Sending..." while processing
  • Button should be disabled during submission
  • No errors should occur

Test Section 2: Verify Success Modal Appears

Step 2.1: Check Success Modal

After submission completes, observe the modal that appears.

Expected:
  • A success modal should appear with a success icon
  • Modal title should be "Task Submitted!"
  • Modal should display the task name: "Test Task - Complete Create Flow"
  • Modal should have an "OK" button to dismiss

Step 2.2: Dismiss the Modal

Click the "OK" button to dismiss the success modal.

Expected: Modal should close and the form should remain visible.

Test Section 3: Verify Form Data Persists

Step 3.1: Check Form Fields After Submission

After dismissing the modal, check all form fields.

Expected:
  • Task Name field should still contain: "Test Task - Complete Create Flow"
  • Description field should still contain: "This is a test task to verify the complete create flow functionality."
  • Priority should still be: High
  • Assigned To should still be: Development Team
  • Created By should still contain: "Test User"
  • Form fields should NOT be cleared

Step 3.2: Check Status Text Area

Check the status text area below the buttons.

Expected: The status text area should be empty (no text-based success messages like "✅ Task submitted successfully").

Test Section 4: Verify Task Appears in History

Step 4.1: Scroll to Task History Section

Scroll down to the "Task History" section below the form.

Expected: The task history section should be visible.

Step 4.2: Verify Task Appears in History

Check if the submitted task appears in the history list.

Expected:
  • The task "Test Task - Complete Create Flow" should appear in the history
  • Task should display all fields: name, description, priority, assigned team, creator
  • Task should show a timestamp
  • Task should have "Edit" and "Delete" buttons
  • Priority badge should be red (High priority)

Step 4.3: Verify Task Order

If there are multiple tasks in history, check the order.

Expected: The most recently submitted task should appear at the top (reverse chronological order).

Test Section 5: Verify "Create Another Task" Button Works

Step 5.1: Check Button Visibility

Scroll back to the form and check for the "Create Another Task" button.

Expected:
  • A green "Create Another Task" button should be visible below the "Submit Task" button
  • Button should be enabled and clickable

Step 5.2: Click "Create Another Task"

Click the "Create Another Task" button.

Expected:
  • All form fields should be cleared to their default values
  • Task Name should be empty
  • Description should be empty
  • Priority should reset to "Low"
  • Assigned To should reset to "Marketing Team"
  • Created By should be empty
  • The "Create Another Task" button should be hidden
  • Submit button should show "Submit Task" (not "Update Task")

Test Section 6: Verify LocalStorage Persistence

Step 6.1: Check LocalStorage

Open browser DevTools (F12), go to Application/Storage tab, and check LocalStorage.

Expected:
  • A key named "taskHistory" should exist in LocalStorage
  • The value should be a JSON object containing a "tasks" array
  • The submitted task should be in the array with all fields including messageId

Step 6.2: Reload Page

Refresh the browser page (F5 or Ctrl+R).

Expected:
  • Page should reload successfully
  • Task history should still display the submitted task
  • All task details should be preserved
